# Экспорт документации из Markdown (локально)

## Важно

- В репозитории храним только Markdown (`.md`) и скрипты.
- Готовые бинарные файлы (PDF/DOCX) **не коммитим в GitHub**.

## Что нужно установить

- `pandoc` (опционально, только для HTML-версии)

## Как запустить экспорт

Из корня проекта:

```bash
./scripts/export-docs.sh
```

Скрипт соберёт документы в папку `build/docs/`.

## Что делает скрипт

- объединяет `README.md`;
- добавляет `docs/playbooks/*.md`;
- добавляет `docs/guides/*.md`;
- добавляет `docs/roadmap/*.md`;
- формирует единый `Markdown bundle` и `manifest`;
- при наличии `pandoc` дополнительно формирует `HTML`.

## Если HTML не собрался

Это не ошибка документации.
Скорее всего не установлен `pandoc`.
Markdown bundle при этом всё равно будет сформирован.

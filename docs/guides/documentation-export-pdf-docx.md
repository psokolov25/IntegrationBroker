# Экспорт документации в PDF и DOCX (локально)

## Важно

- В репозитории храним только Markdown (`.md`) и скрипты.
- Готовые PDF/DOCX **не коммитим в GitHub**.

## Что нужно установить

- `pandoc`
- Для PDF — один из вариантов:
  - `wkhtmltopdf`, или
  - `weasyprint`, или
  - `xelatex`

## Как запустить экспорт

Из корня проекта:

```bash
./scripts/export-docs.sh
```

Скрипт соберёт документы в папку `build/docs/`.

## Что делает скрипт

- объединяет `README.md`;
- добавляет `docs/playbooks/*.md`;
- добавляет `docs/guides/*.md`;
- добавляет `docs/roadmap/*.md`;
- формирует `DOCX` и (если есть PDF-движок) `PDF`.

## Если PDF не собрался

Это не ошибка документации.
Чаще всего просто не установлен PDF-движок.
DOCX при этом будет сформирован.

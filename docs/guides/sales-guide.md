# Руководство для продаж и пресейла

## 1. Что продаём

Integration Broker — это интеграционная платформа для оркестрации потоков между СУО (VisitManager), DataBus, CRM, МИС, системами записи и внешними API.

## 2. Ценность для заказчика

- Сокращает time-to-integration за счёт reusable адаптеров.
- Даёт контролируемую надёжность (idempotency, DLQ, outbox, replay).
- Обеспечивает наблюдаемость и контроль инцидентов.
- Позволяет быстро внедрять клиентские правила через Groovy/YAML без переписывания ядра.

## 3. Ключевые функциональные блоки для презентации

1. Ingress API для входящих сообщений.
2. Оркестрация flow на Groovy.
3. Адаптеры к внешним системам.
4. Надёжность доставки (outbox/DLQ/idempotency).
5. Безопасность (Keycloak, RBAC, санитизация).
6. Frontend Workbench (дорожная карта и старт внедрения).

## 4. Быстрые ответы на частые вопросы

- **Можно ли on-prem?** Да.
- **Можно ли без изменения бизнес-логики ядра?** Да, через flow+Groovy.
- **Можно ли работать с несколькими системами очереди/CRM?** Да, за счёт адаптерного слоя.
- **Как обеспечивается надёжность?** Idempotency + DLQ + outbox + replay.

## 5. Ограничения и честное позиционирование

- IB не заменяет предметные системы (CRM/МИС/СУО), а связывает их.
- Клиентские правила должны жить в сценариях flow/groovy.
- Для высоконагруженных контуров требуется отдельная sizing-сессия.

## 6. Артефакты для сделки

- Архитектурная схема интеграционного контура.
- Матрица интеграций (система -> протокол -> SLA).
- План пилота (2–4 недели) и план промышленного внедрения.

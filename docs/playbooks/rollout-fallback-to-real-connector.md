# Playbook: rollout с fallback на real connector

Краткий чеклист переключения профиля адаптера с fallback-исполнения на нативный коннектор.

## 1) Подготовка

- [ ] Актуализирован runtime-config в git (ревизия + diff).
- [ ] Для коннектора заданы auth/timeout/retry/circuit-breaker.
- [ ] Выполнен `dry-run` для эталонных payload.

## 2) Предрелизная валидация

- [ ] Проверен ответ health-check целевого коннектора.
- [ ] Сравнены результаты `requestedProfile` vs `executionProfile` на smoke-кейсах.
- [ ] Убедились, что `fallback=true` исчезает после переключения.

## 3) Cutover

- [ ] Переключён профиль в runtime-config (`profile` + `connectorId`).
- [ ] Зафиксирован `correlationId` окна переключения для мониторинга.
- [ ] Включён усиленный мониторинг ошибок 4xx/5xx и retry-динамики.

## 4) Stabilization

- [ ] Проверены 3 бизнес-кейса (happy/empty/retryable).
- [ ] Проверен outbox/DLQ на аномальный рост по новому профилю.
- [ ] Подготовлен rollback-план (возврат на fallback-профиль).

## 5) Rollback критерии

- [ ] Ошибки n%+ на критичной операции в течение 15 минут.
- [ ] Несовместимость обязательных полей результата (contract break).
- [ ] Невозможность восстановить поток через replay без потери идемпотентности.
